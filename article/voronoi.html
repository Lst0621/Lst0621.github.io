About Voronoi Diagrams

  • A Voronoi diagram partitions a space into regions, where each region contains all points closer to a particular seed point than to any other seed point.
  • Colored regions: Each color represents a Voronoi cell. All pixels in a colored region are closer to its seed point than to any other seed point.
  • Boundary lines: The black lines are the edges of the Voronoi diagram, where points are equidistant from two or more seed points.
  • Click on the canvas to add new seed points, and right-click anywhere inside a colored region to delete that seed. The diagram will recompute automatically.
  • Use Distance: Euclidean/Torus/Cylinder/Möbius/Klein/Chebyshev/Raw to switch the metric used when assigning cells to seed points:
    • Euclidean: Standard distance metric where cells are determined by the closest seed using straight-line distance.
    • Torus: Periodic distance metric where both horizontal and vertical directions wrap around (like a doughnut surface).
    • Cylinder: Periodic distance metric where only the horizontal direction wraps around, while vertical direction is finite (like a cylinder).
    • Möbius: Periodic in the horizontal direction, but wrapping across the seam flips vertical position (a Möbius strip).
    • Klein: Wraps in both directions, but wrapping across the vertical seam flips horizontal position (a Klein bottle).
    • Chebyshev: L∞ metric where distance is max(|dx|, |dy|) — cells are axis-aligned rectangles determined by coordinate-wise bisectors.
    • Raw: The same Chebyshev construction before the final merge step, useful for inspecting the clipped pieces directly. This is the unmerged Chebyshev raw mode.
  • This visualization uses exact rational arithmetic (Rational<BigInt>) for every clipping and bisector computation, so cell vertices are mathematically exact rationals with no floating-point quantization. The exact numerator/denominator pairs are also exposed via the WASM API (voronoiGetCellBoundariesExact).

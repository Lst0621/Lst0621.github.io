Run Codex and Claude Code in Cursor (with WSL)

Cursor is a VS Code compatible editor, so it can host both the Codex IDE extension and the Claude Code extension. If your repo and build tools live in WSL, open Cursor from the WSL side and let the agents run against the Linux filesystem. Then you can keep separate Codex and Claude Code chats, tabs, and sidebars while they work on the same project.

Why use Cursor as the UI? Today, neither Codex nor Claude Code provides a native Ubuntu desktop app UI. Codex has desktop app support for macOS and Windows, and Claude Code's richer local UI is exposed through editor integrations such as VS Code and Cursor. If your day-to-day environment is Ubuntu inside WSL, the Cursor extension route is the workaround: Cursor supplies the graphical chat, tabs, diff review, and sidebars, while the agents still operate on the Linux repo and tools.
Vocabulary note: Cursor installs these as editor extensions. Codex and Claude Code also have their own plugin systems inside the agents; use those after the extensions are working.

Goal

Windows Cursor window
  - opened from WSL, connected to a Linux repo
  - graphical workaround for missing native Ubuntu desktop UIs
  - Codex panel for OpenAI Codex agent threads
  - Claude Code panel for Claude agent threads
  - separate chats/tabs for different tasks

Prerequisites

  • Windows 10/11 with WSL 2 installed.
  • A repo stored under WSL, for example ~/code/my-project.
  • Cursor installed on Windows.
  • The WSL extension installed in Cursor.
  • A ChatGPT account for Codex, and a Claude account or Anthropic setup for Claude Code.

Setup

1 Open Cursor from WSL

In a WSL terminal, go to your project and launch Cursor:

cd ~/code/my-project
cursor .

If cursor is not available in WSL, open Cursor on Windows, run the command palette with Ctrl+Shift+P, and look for the command that installs the Cursor shell command. You can also open the folder in Cursor and run WSL: Reopen Folder in WSL.

Confirm the window is really in WSL:

echo $WSL_DISTRO_NAME
pwd

You want Linux paths like /home/..., not C:\....

2 Install Codex in Cursor

  1. Open Extensions with Ctrl+Shift+X.
  2. Search for Codex and install the OpenAI Codex extension.
  3. Open the Codex sidebar or run Codex: Open Sidebar from Ctrl+Shift+P.
  4. Sign in when prompted.
  5. Open Cursor settings, search for Codex, and enable running Codex in WSL if your repos live there.

The Codex WSL setting is named chatgpt.runCodexInWindowsSubsystemForLinux. After changing it, reload Cursor so the extension starts in the right environment.

Codex uses the same agent as the Codex CLI and shares configuration through ~/.codex/config.toml. Use the IDE panel for normal work, and use the CLI when you want terminal-first plugin or configuration management.

3 Install Claude Code in Cursor

  1. Open Extensions with Ctrl+Shift+X.
  2. Search for Claude Code and install it for Cursor.
  3. Open the Claude Code panel from the activity bar, status bar, or command palette.
  4. Sign in with your Claude account, or use your configured provider environment.
  5. Send a small prompt, then review the proposed diff before accepting edits.

Claude Code can run as a graphical panel or in terminal mode. Start with the panel so you can use tabs, session history, file mentions, and visible diffs.

4 Manage plugins

For Codex plugins, open Codex CLI inside WSL:

codex
/plugins

Browse the plugin list, install the plugin you need, then start a new Codex thread. Installed Codex plugins can add skills, app integrations, and MCP servers.

For Claude Code plugins, use the Claude prompt box:

/plugins

The Claude Code plugin dialog lets you install available plugins, enable or disable installed plugins, and add plugin marketplaces. Restart Claude Code when it asks you to apply marketplace changes.

5 Use separate chats for separate jobs

  • Use Codex for tasks where you want OpenAI Codex, repo-aware edits, command execution, or Codex plugins.
  • Use Claude Code for tasks where you want Claude's plan review, Claude sessions, or Claude plugins.
  • Open new tabs or panels for unrelated tasks so each agent keeps a clean local context.
  • Name the chats by task, for example fix-login-test, draft-doc-page, or review-pr.
  • Keep the repo under WSL's Linux filesystem for faster file I/O and fewer permission surprises.

Troubleshooting

Cursor opened the Windows copy of the repo

Reopen the folder in WSL from the command palette, or close Cursor and launch it again from the WSL terminal with cursor ..

The agent cannot find Linux tools

Make sure the integrated terminal is a WSL terminal and that the extension is running in the WSL remote window. Run which node, which git, or your project build command from that terminal.

A plugin installed but does not appear

Start a new agent thread. If you changed marketplaces or local plugin files, restart the agent or reload the Cursor window.

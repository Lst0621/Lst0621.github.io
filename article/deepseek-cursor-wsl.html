Use DeepSeek in Cursor (with WSL + Cloudflare Tunnel)

A practical guide to running DeepSeek's latest reasoning models (deepseek-v4-pro / deepseek-v4-flash) inside Cursor on Windows via WSL, using deepseek-lane as a local proxy and Cloudflare Tunnel for public HTTPS exposure.

Why You Need a Proxy

DeepSeek's reasoning models require that every turn in a multi-turn conversation includes the full chain of reasoning_content. Cursor doesn't pass this field back on tool‑call follow‑up requests, so the upstream API returns:

Provider returned error: reasoning_content must be passed back to the API.

deepseek-lane sits between Cursor and the DeepSeek API. It caches reasoning tokens in a local SQLite database and re‑injects them when Cursor sends a follow‑up request — making DeepSeek work flawlessly inside Cursor.

It also:

  • Wraps the model's thinking in collapsible Markdown blocks so you can read the reasoning inside Cursor.
  • Normalises old‑style functions/function_call fields to the modern tools/tool_choice format.
  • Provides a public HTTPS tunnel (so Cursor's non‑localhost restriction is satisfied).

Architecture Overview

Cursor  ──▶  Cloudflare Tunnel (public HTTPS)  ──▶  deepseek-lane (127.0.0.1:19199)  ──▶  DeepSeek API
                https://xxx.trycloudflare.com/v1              reasoning cache + transform

Prerequisites

  • Windows 10/11 with WSL 2 enabled.
  • Node.js 20+ installed inside WSL.
  • A DeepSeek API key (from DeepSeek Platform or an opencode subscription).
  • Cursor installed on Windows.
Tip: If you don't have Node.js yet, install it via nvm: 
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
nvm install 22

Step‑by‑Step Setup

1 Install deepseek-lane

Inside your WSL terminal:

npm install -g deepseek-lane

Verify the installation:

dsl --version

2 Install Cloudflare Tunnel (cloudflared)

Cursor does not allow localhost or other non‑public API addresses. Cloudflare Tunnel gives us a free public HTTPS URL without opening router ports. 

wget https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb
sudo dpkg -i cloudflared-linux-amd64.deb

Check that it works:

cloudflared --version
No Cloudflare account needed. cloudflared tunnel --url works with TryCloudflare — a free, instant tunnel with no sign‑up.

3 Start the Proxy (Without Tunnel First)

The first run launches an interactive wizard. Answer the prompts: 

dsl start --no-interactive \
  --base-url https://api.deepseek.com/v1 \
  --model deepseek-v4-pro \
  --thinking enabled \
  --reasoning-effort max \
  --port 19199 \
  --no-ngrok
FlagWhat it does
--base-urlYour DeepSeek API endpoint. Use https://api.deepseek.com/v1 for the official API, or your opencode endpoint.
--modeldeepseek-v4-pro (best reasoning) or deepseek-v4-flash (faster).
--thinking enabledEnables DeepSeek's reasoning / thinking mode.
--reasoning-effort maxMaximum reasoning depth. Options: low, medium, high, max, xhigh.
--port 19199Local port the proxy listens on.
--no-ngrokSkip ngrok — we use Cloudflare Tunnel instead.

After it starts you'll see the local address. Keep this terminal running. 

✓ Model: deepseek-v4-pro (thinking, max)
▸ Local:  http://127.0.0.1:19199/v1

4 Start Cloudflare Tunnel

Open a second WSL terminal and run:

cloudflared tunnel --url http://127.0.0.1:19199

You'll see output like:

2026-05-18T22:30:00Z INF Requesting new quick Tunnel on trycloudflare.com...
2026-05-18T22:30:02Z INF +---------------------------------------------+
2026-05-18T22:30:02Z INF |  Your quick Tunnel has been created!        |
2026-05-18T22:30:02Z INF |  https://happy-foxes-trycloudflare.com    |
2026-05-18T22:30:02Z INF +---------------------------------------------+

Copy the https://*.trycloudflare.com URL — you'll paste it into Cursor in the next step.

Tip: The tunnel URL changes every time you restart cloudflared. Keep the tunnel terminal open while using Cursor. If it disconnects, restart it and update the URL in Cursor.

5 Configure Cursor

Now switch to Cursor on Windows:

  1. Open Settings — Ctrl+Shift+J (or Cmd+Shift+J on Mac), then Models tab.
  2. Uncheck (disable) all built‑in models so Cursor only uses your custom one.
  3. Under OpenAI API Key, paste your DeepSeek API key.

    (Yes — you put the DeepSeek key into the OpenAI key field. The proxy forwards requests to DeepSeek regardless.)

  4. Click Add Model and fill in:
    Model namedeepseek-v4-pro
    Base URLhttps://happy-foxes-trycloudflare.com/v1

    Replace the URL with the one from your tunnel. Make sure it ends with /v1.

  5. Press Ctrl+Shift+0 (or Cmd+Shift+0) to toggle "Override OpenAI Base URL" — make sure it's enabled.
Model name must match exactly. Use the same model name in Cursor that you passed to dsl start --model <name>.

Complete Workflow (Cheat Sheet)

Every time you want to use DeepSeek in Cursor:

  1. WSL terminal 1: 
    dsl start --port 19199 --no-ngrok --model deepseek-v4-pro
  2. WSL terminal 2: 
    cloudflared tunnel --url http://127.0.0.1:19199
  3. Copy the https://*.trycloudflare.com URL.
  4. Cursor: update the model's Base URL to https://*.trycloudflare.com/v1 if it changed.
  5. Select deepseek-v4-pro and start chatting.
